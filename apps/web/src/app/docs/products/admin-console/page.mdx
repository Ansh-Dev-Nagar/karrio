# Admin Console

Karrio's Admin Console provides a comprehensive administrative interface for managing users, organizations, system settings, and platform configuration. Access advanced features for system monitoring, user management, and operational oversight.

## Overview

The Admin Console is designed for system administrators and platform managers who need to oversee the entire Karrio deployment. Manage users across organizations, configure system-wide settings, and monitor platform health and performance.

### Key Features

- **User Management**: Manage users across all organizations
- **Organization Administration**: Create and manage organizations
- **System Configuration**: Configure platform-wide settings
- **Monitoring & Analytics**: Monitor system health and usage
- **Security Management**: Manage security policies and access controls

## Access Control

### Admin Roles

The Admin Console supports different levels of administrative access:

#### Super Admin

- Full system access
- User and organization management
- System configuration
- Security policy management

#### Organization Admin

- Manage users within assigned organizations
- Configure organization settings
- Access organization analytics

#### Support Admin

- Read-only access to user data
- Access to support tools
- Limited user assistance capabilities

## GraphQL Admin Operations

The Admin Console provides GraphQL-based administration through the `/graphql` endpoint with admin authentication. All operations require proper admin privileges.

### Available Admin Queries

- `configs` - Get system configuration
- `system_usage` - Get platform usage statistics
- `system_connections` - List system carrier connections
- `system_carrier_connections` - List system carriers with pagination

### Available Admin Mutations

- `update_configs` - Update system configuration flags
- `create_system_carrier_connection` - Create system-wide carrier
- `update_system_carrier_connection` - Update system carrier
- `delete_system_carrier_connection` - Remove system carrier

### JavaScript SDK Usage

```javascript
import { GraphQLApi } from "@karrio/types";

const admin = new GraphQLApi("https://api.karrio.io", client.axios);

// Get system config
const config = await admin.request(`
  query {
    configs {
      ALLOW_SIGNUP
      MULTI_ORGANIZATIONS
      ORDERS_MANAGEMENT
      DOCUMENTS_MANAGEMENT
    }
  }
`);

// Update configuration
const update = await admin.request(
  `
  mutation updateConfigs($input: InstanceConfigMutationInput!) {
    update_configs(input: $input) {
      configs {
        ALLOW_SIGNUP
        MULTI_ORGANIZATIONS
      }
    }
  }
`,
  {
    input: {
      ALLOW_SIGNUP: false,
      MULTI_ORGANIZATIONS: true,
    },
  },
);
```

## Best Practices

### Security Guidelines

1. **Use Strong Authentication**: Require MFA for all admin accounts
2. **Principle of Least Privilege**: Grant minimum necessary permissions
3. **Regular Audits**: Review admin access and activities regularly
4. **Secure Communication**: Always use HTTPS for admin operations

### Operational Procedures

```bash
# Regular maintenance checklist:
# 1. Review system health and performance metrics
# 2. Check audit logs for suspicious activities
# 3. Update security policies as needed
# 4. Monitor carrier integrations and performance
# 5. Review user and organization growth
# 6. Backup critical data regularly
```
